# Web Mapping Fundamentals

## Objective

Understand the structure of a web application before performing any security testing.

---

## Application Pages

- Home Page
- Blog Post Page
- Login Page

---

## Features Identified

### Blog System

- View blog posts
- Display author information
- Submit comments

### Authentication System

- User login functionality

---

## Parameters Observed

### Blog Post

Parameter:

postId

Example:

/post?postId=5

Purpose:

Used by the application to determine which blog post should be displayed.

---

## Input Points Identified

### Login Form

- username
- password

### Comment Form

- Name
- Email
- Website
- Comment

---

## User Interaction Flow

Visitor
↓
Home Page
↓
Select Blog Post
↓
Read Article
↓
View Author Information
↓
Submit Comment

OR

Visitor
↓
Login Page
↓
Enter Credentials
↓
Authenticate

---

## Key Learning Points

- Identify application pages
- Identify features and functionality
- Identify user input locations
- Identify URL parameters
- Understand user workflows
- Build an application map before security testing

---

## Conclusion

Before testing security, it is important to understand how the application works, where data is entered, how users interact with the system, and what functionality is available.
